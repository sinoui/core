import SvgIcon from '@sinoui/core/SvgIcon';

## SvgIcon

```tsx
import SvgIcon from '@sinoui/core/SvgIcon';

function SvgIconDemo() {
  return (
    <SvgIcon>
      <path d="M19 6.41L17.59 5 12 10.59 6.41 5 5 6.41 10.59 12 5 17.59 6.41 19 12 13.41 17.59 19 19 17.59 13.41 12z" />
    </SvgIcon>
  );
}
```

## 内置图标组件

[@sinoui/icons](https://github.com/sinoui/icons)  提供了  1100+  个  [Material Design  图标](https://material.io/resources/icons/?style=baseline)。更多详细使用方式可参考  [@sinoui/icons  官方文档](https://sinoui.github.io/icons)。

因为  @sinoui/icons  中提供的每个图标组件，都已经使用了  SvgIcon  组件包装，所以在使用组件时，不需要再使用  `<SvgIcon>`  包裹：

```tsx
import CloseIcon from '@sinoui/icons/Close';

function SvgIconDemo() {
  return <CloseIcon />;
}
```

不需要这么做：

```tsx
import SvgIcon from '@sinoui/core/SvgIcon';
import CloseIcon from '@sinoui/icons/Close';

function SvgIconDemo() {
  // ️⚠️ 不需要用 SvgIcon 包裹 @sinoui/icons 中的图标组件
  return (
    <SvgIcon>
      <CloseIcon />
    </SvgIcon>
  );
}
```

## 与第三方图标库结合使用

```tsx
import SvgIcon from '@sinoui/core/SvgIcon';
import { WiDayCloudyGusts } from 'react-icons/wi';

const SvgIconDemo = () => <SvgIcon as={WiDayCloudyGusts} />;
```

效果如下：

import { WiDayCloudyGusts } from 'react-icons/wi';

<StoryLayout>
  <SvgIcon as={WiDayCloudyGusts} />
</StoryLayout>

有时，第三方图标库提供的 Svg 图标的`viewBox`不是默认的`0 0 24 24`，且图标库已经包含合适的`viewBox`时，可以使用`SvgIcon`组件的`disabledViewBox`属性禁止`SvgIcon`向图标组件传递`viewBox`属性，如下所示：

```tsx
import SvgIcon from '@sinoui/core/SvgIcon';
import { FaWpforms } from 'react-icons/fa';

const SvgIconDemo = () => <SvgIcon as={FaWpforms} disabledViewBox />;
```

效果如下：

import { FaWpforms } from 'react-icons/fa';

<StoryLayout>
  <SvgIcon as={FaWpforms} disabledViewBox />
</StoryLayout>

## 引用 svg 文件

通过 svgr 可在打包时自动将 svg 文件转换成 React 组件，在 webpack.config.js 中的配置如下：

```tsx
{
  test: /\.svg$/,
  use: ['@svgr/webpack'],
}
```

在代码中的使用方式：

```tsx
import StarIcon from './star.svg';

<StoryLayout>
  <SvgIcon as={StarIcon} color="actionActive" />
</StoryLayout>;
```

效果如下：

<StoryLayout>
  <SvgIcon as={StarIcon} color="actionActive" />
</StoryLayout>
